# 2025 E 基础部分（2）（3）：瞄准解算与 PyBullet 仿真

## 1. 三层职责

本项目把问题拆成三层，避免把视觉、控制和仿真写进同一个正式模块。

1. **视觉层**：检测 A4 靶纸黑框，输出按左上、右上、右下、左下排列的四个像素角点。
2. **坐标解算层**：使用相机内参、畸变参数和 A4 实际尺寸，通过 `solvePnP` 求靶心相对相机的三维坐标，再换算为云台 yaw/pitch 增量。
3. **任务与验证层**：基础（2）直接检测并瞄准；基础（3）在看不到靶时先扫描。PyBullet 只负责提供可重复的虚拟场景、目标投影和激光落点判定，不替代树莓派上的 OpenCV。

数据流如下：

```mermaid
flowchart LR
    A["摄像头图像"] --> B["黑框检测与跟踪"]
    B --> C["四角点 TL/TR/BR/BL"]
    C --> D["solvePnP 求靶心三维坐标"]
    D --> E["补偿激光与相机的位置/角度偏差"]
    E --> F["yaw、pitch 增量"]
    F --> G["串口发送给两轴电机控制器"]
    G --> H["连续稳定后触发激光"]
```

## 2. 激光笔解算逻辑

### 2.1 输入

- 黑框四角点，顺序为 TL、TR、BR、BL；
- 相机内参矩阵和畸变系数；
- 横放 A4 靶纸物理尺寸：宽 `0.297 m`、高 `0.210 m`；
- 激光发射点相对相机光心的位置偏移；
- 激光轴线相对相机光轴的固定角度偏差。

### 2.2 从二维角点求三维靶心

给 A4 的四个物理角点建立靶纸坐标系，`solvePnP` 求得旋转 `R` 和位移 `t`。靶纸坐标原点设在中心，因此 `t=(X,Y,Z)` 就是靶心在相机坐标系中的位置，其中 X 向右、Y 向下、Z 向前。

完整图像无需先做一次昂贵的 `remap`。`solvePnP` 可以直接接收畸变系数，只处理四个角点，计算量更小。

### 2.3 补偿激光与相机不共轴

若激光发射点在相机坐标系中为 `L`，靶心为 `T`，实际需要的射线方向为：

`d = T - L`

再由方向向量得到：

- `yaw = atan2(dx, dz)`；
- `pitch = atan2(-dy, sqrt(dx² + dz²))`。

最后减去实测的激光轴固定角度偏差。该步骤能补偿近距离下明显的视差；只用画面中心像素不能完成这种补偿。

解算器最终只输出两个增量值，不直接操作 GPIO 或串口：

- yaw 正值：向右转；
- pitch 正值：向上转。

步进电机控制层还需把角度换算成位置/脉冲或 PD42S1 协议命令，并限制速度、加速度和机械范围。

## 3. 基础部分（2）

题目要求：小车在场地中，位置和姿态自定；启动瞄准模块后 2 秒内发射并击中靶心，`D1 <= 2 cm`。

建议状态流程：

1. 预先让相机大致朝向靶纸，使完整黑框进入视野；
2. 检测并跟踪黑框；
3. 解算 yaw/pitch 增量；
4. 电机转动后重复检测和解算，形成视觉闭环；
5. 连续若干帧误差进入死区后才允许发射；
6. 超过 2 秒仍未锁定则本轮失败，不盲目发射。

这里“位置姿态自定”是基础（2）的优势，可以通过摆车和机械初始角度减少搜索时间。

## 4. 基础部分（3）

题目要求：小车位于轨迹指定点，瞄准模块初始方向任意；启动后 4 秒内自动瞄准并发射，`D1 <= 2 cm`。

它比基础（2）多一个搜索阶段：

1. `SEARCH`：未发现完整黑框时，yaw 按固定方向扫描，pitch 回到覆盖靶纸高度的搜索角；
2. `AIM`：发现黑框后停止匀速扫描，改用视觉解算增量闭环逼近；
3. `LOCK`：连续多帧误差合格，避免单帧误检触发；
4. `FIRE`：向下位机发一次触发指令；
5. `TIMEOUT`：4 秒内没有锁定则停止发射。

扫描速度必须由实测视场角和电机可用速度决定。仿真中的 `180°/s` 只是当前假设，不应直接当成 PD42S1 最终参数。

## 5. PyBullet 仿真做了什么

`scripts/test/simulate_basic_aiming.py` 建立 1 m × 1 m 方形轨迹、A4 靶板、小车、相机和带安装误差的激光射线。每个控制周期执行：

1. 根据虚拟相机姿态把 A4 四角投影成像素坐标；
2. 加入可调像素噪声；
3. 将四角点交给正式 `LaserAimSolver`；
4. 按假设的云台速度限制执行 yaw/pitch 增量；
5. 用 `rayTest` 求激光与靶板交点；
6. 计算交点到靶心的距离 `D1`；
7. 连续 3 帧满足 `D1 <= 2 cm` 才判定成功，并检查总时间。

仿真验证的是“几何关系和状态流程在给定假设下是否自洽”。它尚未真实模拟以下因素：电机通信延迟、机械回差、车体振动、滚动快门、曝光变化、黑框误检、激光光斑检测和 PD42S1 的实际速度曲线。因此仿真通过不等于实物必然通过，最终仍需硬件标定和录像/实机统计。
