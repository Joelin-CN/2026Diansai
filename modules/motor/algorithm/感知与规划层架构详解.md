# 感知与规划层架构详解

感知与规划层是智能小车的上层系统，负责回答三个问题：

1. 小车在哪里？
2. 周围环境是什么样的？
3. 接下来应该沿什么轨迹运动？

该层最终向运动控制层输出带时间信息的参考轨迹，例如：

$$
\boldsymbol{x}_{ref}(t) = [x(t), y(t), \theta(t), v(t), \omega(t), a(t), \alpha(t)]^T
$$

其中，$x,y,\theta$ 表示期望位姿，$v,\omega$ 表示期望线速度和角速度，$a,\alpha$ 表示期望线加速度和角加速度。

---

## 一、整体架构

```text
传感器原始数据
      │
      v
┌──────────────┐
│ 感知与预处理  │ 摄像头、雷达、红外、编码器、IMU
└──────┬───────┘
       v
┌──────────────┐
│ 状态估计与建图│ 当前位姿、速度、障碍物、地图
└──────┬───────┘
       v
┌──────────────┐
│ 行为与任务决策│ 跟线、避障、停车、到达目标
└──────┬───────┘
       v
┌──────────────┐
│ 路径规划      │ 生成不含时间信息的几何路径
└──────┬───────┘
       v
┌──────────────┐
│ 轨迹规划      │ 加入速度、加速度和时间约束
└──────┬───────┘
       v
参考轨迹 x_ref(t)
       │
       v
运动控制层（MPC、LQR、PID、Pure Pursuit等）
```

各部分的职责如下：

| 模块 | 输入 | 输出 | 核心任务 | 典型频率 |
|------|------|------|----------|----------|
| 传感器预处理 | 原始图像、点云、脉冲、IMU数据 | 去噪和标定后的观测量 | 滤波、同步、标定 | 50 Hz～1 kHz |
| 感知 | 预处理后的观测量 | 车道、障碍物、标志物 | 环境理解 | 10～60 Hz |
| 状态估计 | 编码器、IMU、视觉观测 | 位姿、速度及其不确定度 | 多传感器融合 | 50～500 Hz |
| 行为决策 | 任务目标、环境状态 | 当前行为模式 | 任务分解和状态切换 | 5～20 Hz |
| 路径规划 | 起点、终点、地图 | 几何路径 | 避障和可达性搜索 | 1～10 Hz或事件触发 |
| 轨迹规划 | 几何路径、执行器约束 | 时间参数化轨迹 | 平滑化和速度规划 | 10～100 Hz |

感知、规划和控制不是严格串行的。工程系统通常让它们以不同频率并行运行，并通过带时间戳的数据或消息队列交换结果。

## 二、感知层

### 2.1 传感器配置

电赛小车常用传感器如下：

| 传感器 | 主要作用 | 数据类型 | 特点 |
|--------|----------|----------|------|
| 摄像头 | 路径、色块、二维码、AprilTag、障碍物识别 | 图像 | 信息丰富，但受光照和算力影响 |
| 激光雷达 | 障碍物测距和建图 | 点云或距离扫描 | 距离稳定，但成本和处理量较高 |
| 超声波 | 近距离避障 | 距离 | 成本低，但波束宽、刷新率较低 |
| 红外阵列 | 巡线、边缘检测 | 数字量或模拟量 | 低延迟、适合规则赛道 |
| 编码器 | 轮速和里程测量 | 脉冲计数 | 短期精度高，但会累积误差 |
| IMU | 角速度和加速度测量 | 六轴或九轴数据 | 高频，但存在噪声和零偏漂移 |
| 磁力计 | 绝对航向辅助 | 三轴磁场 | 容易受到电机和金属结构干扰 |

### 2.2 传感器预处理

原始数据不能直接用于规划，通常需要经过以下处理：

1. 标定传感器内参和外参。
2. 将不同传感器数据统一到同一坐标系。
3. 使用时间戳完成数据同步。
4. 去除异常值和高频噪声。
5. 估计观测量的不确定度。

坐标系通常至少包括：

- `world/map`：固定世界坐标系。
- `odom`：局部连续的里程计坐标系。
- `base_link`：小车底盘坐标系。
- `camera`、`imu`、`lidar`：各传感器坐标系。

若传感器外参不准确，算法可能在静态情况下正常，但小车转弯时会产生系统性误差。因此，坐标变换和时间同步是感知架构的基础，而不是后期优化项。

### 2.3 定位与状态估计

定位回答“我在哪里”，状态估计则进一步给出速度、角速度和估计置信度。常用状态向量为：

$$
\boldsymbol{x}=[x,y,\theta,v,\omega,b_g]^T
$$

其中 $b_g$ 是陀螺仪零偏。

常见定位方法：

| 方法 | 原理 | 优点 | 缺点 |
|------|------|------|------|
| 航位推算 | 积分编码器和IMU数据 | 高频、实现简单 | 误差持续累积 |
| 地标定位 | 识别AprilTag、二维码或固定特征 | 可提供绝对修正 | 依赖可见地标 |
| 视觉里程计 | 跟踪图像特征估计运动 | 不依赖轮胎接触 | 受纹理和光照影响 |
| SLAM | 同时建立地图和定位 | 适合未知环境 | 算力和调试成本较高 |
| UWB/GPS | 外部绝对定位 | 无累计漂移 | 需要基础设施或室外信号 |

典型EKF融合流程：

```python
# 预测：使用运动学模型、编码器和IMU
x_pred = f(x_prev, control, dt)
P_pred = F @ P_prev @ F.T + Q

# 更新：使用视觉地标、UWB或其他绝对观测
innovation = measurement - h(x_pred)
S = H @ P_pred @ H.T + R
K = P_pred @ H.T @ inv(S)
x_est = x_pred + K @ innovation
P_est = (I - K @ H) @ P_pred
```

这里的 $P$ 是状态协方差，$Q$ 是过程噪声协方差，$R$ 是观测噪声协方差。规划器不应只接收一个“绝对正确”的位姿，也应知道估计是否可靠，以便在定位不确定时主动降速或停车。

### 2.4 环境感知

环境感知回答“周围有什么”，典型任务包括：

- 路径或车道中心线检测。
- 障碍物检测和跟踪。
- 目标、颜色、标志物和二维码识别。
- 可行驶区域分割。
- 停车点或操作区域检测。

算法选择应从任务约束出发：

| 方法 | 速度 | 算力需求 | 适用场景 |
|------|------|----------|----------|
| 阈值、形态学、边缘和霍夫变换 | 高 | 低 | 规则赛道、固定颜色和几何特征 |
| 模板或特征匹配 | 中高 | 低到中 | 二维码、AprilTag、固定目标 |
| 轻量神经网络 | 中 | 中 | 目标外观变化较大的场景 |
| YOLO或语义分割网络 | 中低 | 高 | 多目标和复杂背景 |

规则明确的电赛场地通常优先使用传统视觉，因为它延迟低、行为可解释、调试成本小。只有传统方法无法覆盖环境变化时，再引入神经网络。

一个简单的视觉巡线流程如下：

```python
def detect_line(image):
    roi = image[image.shape[0] // 2:, :]
    gray = cv2.cvtColor(roi, cv2.COLOR_BGR2GRAY)
    binary = cv2.adaptiveThreshold(
        gray, 255, cv2.ADAPTIVE_THRESH_GAUSSIAN_C,
        cv2.THRESH_BINARY, 21, 5
    )

    contours, _ = cv2.findContours(
        binary, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE
    )
    if not contours:
        return None

    contour = max(contours, key=cv2.contourArea)
    moments = cv2.moments(contour)
    if moments["m00"] == 0:
        return None

    center_x = moments["m10"] / moments["m00"]
    lateral_error = center_x - roi.shape[1] / 2
    return lateral_error
```

### 2.5 环境建模

感知结果需要转换成规划器可使用的环境模型：

| 地图类型 | 数据结构 | 适用任务 |
|----------|----------|----------|
| 占据栅格地图 | 二维概率数组 | A*、Dijkstra和避障 |
| 代价地图 | 每个栅格保存通行代价 | 给障碍物留安全距离 |
| 拓扑地图 | 节点和边组成的图 | 固定站点导航和任务调度 |
| 特征地图 | 地标位置和描述符 | 视觉定位 |
| 语义地图 | 对象类别、状态和位置 | 任务理解与行为规划 |

规划时通常不直接使用障碍物原始边界，而是根据小车外形和安全余量对障碍物进行膨胀。这样可以将有尺寸的小车近似为一个点进行路径搜索。

## 三、行为与任务决策层

行为决策层负责把比赛任务转换为当前应该执行的行为，例如：

- 等待启动。
- 正常巡线。
- 接近目标。
- 避障。
- 精确停车。
- 执行机械动作。
- 任务完成或故障停车。

对于规则清晰的电赛任务，有限状态机通常比复杂决策模型更合适：

```python
class TaskStateMachine:
    def __init__(self):
        self.state = "IDLE"

    def update(self, observation):
        if self.state == "IDLE" and observation.start:
            self.state = "FOLLOW_PATH"

        elif self.state == "FOLLOW_PATH":
            if observation.obstacle:
                self.state = "AVOID_OBSTACLE"
            elif observation.target_detected:
                self.state = "APPROACH_TARGET"

        elif self.state == "AVOID_OBSTACLE":
            if observation.path_clear:
                self.state = "FOLLOW_PATH"

        elif self.state == "APPROACH_TARGET":
            if observation.target_distance < 0.1:
                self.state = "STOPPED"

        return self.state
```

状态机必须明确规定：

- 每个状态的进入条件。
- 每个状态的退出条件。
- 超时处理。
- 感知丢失时的降级策略。
- 紧急停止的最高优先级。

当任务数量较多、行为需要复用时，可以改用行为树。行为树比状态机更容易组合“条件检查、顺序执行、失败回退”等逻辑。

## 四、路径规划层

路径是空间中的几何曲线，不一定包含速度和时间信息。例如：

$$
\mathcal{P} = \{(x_0,y_0),(x_1,y_1),\ldots,(x_n,y_n)\}
$$

路径规划的目标是在地图上找到一条无碰撞、代价较低且可被底盘跟踪的路径。

### 4.1 全局路径规划

全局规划使用完整或大范围地图，从当前位置规划到任务目标。常用算法包括：

| 算法 | 特点 | 适用场景 |
|------|------|----------|
| Dijkstra | 可获得最短路径，不使用启发函数 | 地图较小或需要完整距离场 |
| A* | 使用启发函数提高搜索速度 | 二维栅格地图的首选方案 |
| Theta* | 允许任意角度连接 | 减少A*的折线路径 |
| D* Lite | 地图变化时增量重规划 | 动态发现障碍物 |
| RRT/RRT* | 随机采样，容易处理复杂约束 | 连续空间或高维系统 |

A*的评价函数为：

$$
f(n)=g(n)+h(n)
$$

其中 $g(n)$ 是起点到节点 $n$ 的实际代价，$h(n)$ 是节点到终点的估计代价。

```python
import heapq

def astar(grid, start, goal):
    open_set = [(0.0, start)]
    came_from = {}
    g_score = {start: 0.0}

    while open_set:
        _, current = heapq.heappop(open_set)
        if current == goal:
            return reconstruct_path(came_from, current)

        for neighbor in get_neighbors(current):
            if grid[neighbor] == 1:
                continue

            candidate = g_score[current] + distance(current, neighbor)
            if candidate >= g_score.get(neighbor, float("inf")):
                continue

            came_from[neighbor] = current
            g_score[neighbor] = candidate
            priority = candidate + distance(neighbor, goal)
            heapq.heappush(open_set, (priority, neighbor))

    return None
```

### 4.2 局部路径规划

局部规划只关注小车附近的一段空间，用于绕开新出现的障碍物，并尽量回到全局路径。常见算法包括：

- DWA：在 $(v,\omega)$ 速度空间内采样并预测短期轨迹。
- TEB：优化带时间间隔的轨迹点，同时考虑障碍物和运动学约束。
- 人工势场：目标产生引力，障碍物产生斥力。
- 局部MPC：直接在预测窗口内优化状态与控制输入。

DWA会对每个候选控制量进行前向模拟：

$$
\dot{x}=v\cos\theta,\quad
\dot{y}=v\sin\theta,\quad
\dot{\theta}=\omega
$$

并综合评价：

$$
J=w_gJ_{goal}+w_pJ_{path}+w_oJ_{obstacle}+w_vJ_{velocity}
$$

```python
def dwa_plan(state, goal, obstacles, limits, dt=0.05):
    best_score = float("inf")
    best_command = (0.0, 0.0)

    for v in velocity_samples(state.v, limits, dt):
        for omega in angular_samples(state.omega, limits, dt):
            trajectory = simulate(state, v, omega, horizon=1.5, dt=dt)
            if collides(trajectory, obstacles):
                continue

            score = (
                goal_cost(trajectory, goal)
                + path_cost(trajectory)
                + obstacle_cost(trajectory, obstacles)
                + speed_cost(v)
            )
            if score < best_score:
                best_score = score
                best_command = (v, omega)

    return best_command
```

人工势场实现简单，但容易陷入局部极小值，因此更适合作为简单避障修正，而不是复杂环境中的唯一规划器。

### 4.3 路径平滑

栅格搜索得到的路径通常含有折角。如果直接跟踪，会造成角速度突变和轮速冲击。因此必须经过：

1. 删除冗余路径点。
2. 视线检查或捷径优化。
3. 样条、Bezier或Clothoid曲线拟合。
4. 曲率约束检查。

对于平面路径 $(x(s),y(s))$，曲率为：

$$
\kappa(s)=\frac{x'(s)y''(s)-y'(s)x''(s)}{\left(x'(s)^2+y'(s)^2\right)^{3/2}}
$$

差速底盘满足：

$$
\omega=v\kappa
$$

因此，即使路径在几何上连续，如果曲率突变，角速度仍会突变。为了平滑控制，应优先追求曲率连续，而不只是位置连续。

## 五、轨迹规划层

路径只描述“从哪里经过”，轨迹还要描述“什么时候经过”和“以多快速度经过”。轨迹规划会在路径上添加时间参数：

$$
\boldsymbol{p}(t)=[x(t),y(t),\theta(t)]^T
$$

并满足执行器约束：

$$
|v|\le v_{max},\quad
|\omega|\le \omega_{max},\quad
|a|\le a_{max},\quad
|\alpha|\le \alpha_{max},\quad
|j|\le j_{max}
$$

### 5.1 基于曲率的速度限制

路径越弯，小车允许的速度越低。若限制横向加速度 $a_y$，则：

$$
a_y=v^2\kappa
$$

因此：

$$
v_{curve}(s)\le\sqrt{\frac{a_{y,max}}{|\kappa(s)|}}
$$

规划器可以先根据曲率计算每个路径点的速度上限，再通过前向和反向扫描施加加速与减速约束。

### 5.2 梯形与S曲线速度规划

梯形速度规划限制最大速度和加速度，但加速度会发生阶跃，意味着jerk理论上无穷大。S曲线进一步限制jerk：

$$
j(t)=\frac{da(t)}{dt}
$$

典型七段式S曲线包括：

1. jerk为正，加速度上升。
2. 加速度保持正最大值。
3. jerk为负，加速度下降到零。
4. 匀速。
5. jerk为负，减速度幅值上升。
6. 保持最大减速度。
7. jerk为正，减速度回到零。

S曲线可以减小电机电流冲击、车体俯仰、机械间隙碰撞和轮胎打滑，是实现“平滑运动”的关键环节之一。

### 5.3 轨迹重规划

局部轨迹需要周期性重规划，但每次重规划不能让输出突变。新轨迹起点应使用当前估计状态，至少保证：

- 位置连续。
- 速度连续。
- 尽量保证加速度连续。

如果每次规划都从零速度开始，系统会产生周期性顿挫。正确做法是将当前 $v,\omega,a$ 作为新轨迹的初始边界条件。

## 六、感知、规划和控制的接口

### 6.1 推荐的数据接口

感知层输出：

```c
typedef struct {
    uint64_t timestamp_us;
    float x;
    float y;
    float theta;
    float linear_velocity;
    float angular_velocity;
    float covariance[25];
    bool localization_valid;
} VehicleState;
```

轨迹规划层输出：

```c
typedef struct {
    uint64_t timestamp_us;
    float x;
    float y;
    float theta;
    float linear_velocity;
    float angular_velocity;
    float linear_acceleration;
    float angular_acceleration;
    float curvature;
} TrajectoryPoint;
```

运动控制层不应直接消费“摄像头偏差”这类未经定义的原始量，而应尽量接收有明确坐标系、时间戳和单位的状态或轨迹数据。

### 6.2 多频率并行运行

典型调度可以是：

| 任务 | 频率 |
|------|------|
| 电流环 | 10～20 kHz |
| 电机速度环 | 500 Hz～2 kHz |
| IMU采样与状态估计 | 100～500 Hz |
| 底盘轨迹跟踪 | 50～200 Hz |
| 图像感知 | 20～60 Hz |
| 局部轨迹规划 | 10～30 Hz |
| 全局规划和任务决策 | 1～10 Hz或事件触发 |

控制器应使用轨迹时间戳进行插值，而不是简单地重复使用规划器最后一个输出点。否则规划频率较低时，参考量会呈阶梯变化。

### 6.3 延迟补偿

摄像头处理通常会带来几十毫秒延迟。若图像在 $t_m$ 时刻曝光，但在 $t_n$ 时刻才完成处理，则识别结果对应的是历史状态。应使用时间戳，将观测更新到 $t_m$，然后再通过运动模型预测到当前时刻 $t_n$。

高速小车中，延迟常常比传感器噪声更危险。例如速度为 $2\,m/s$、总延迟为 $50\,ms$ 时，小车在算法输出前已经前进了 $0.1\,m$。

## 七、性能与安全设计

### 7.1 实时性

每个模块都应设置计算预算：

| 模块 | 建议策略 |
|------|----------|
| 图像处理 | 限定ROI、降低分辨率、减少内存复制 |
| 全局规划 | 事件触发、复用旧路径、限制搜索范围 |
| 局部规划 | 固定采样数量和预测窗口 |
| 状态估计 | 固定维度矩阵、避免动态内存分配 |

规划器超时不能阻塞底层控制。超时时应继续执行上一条仍然有效的轨迹，并在超过安全时限后减速停车。

### 7.2 数据有效性和降级策略

系统必须区分：

- 没有检测到目标。
- 检测到了目标但置信度较低。
- 数据已经超时。
- 传感器硬件故障。

建议的降级策略：

1. 单帧视觉丢失：短时间使用状态预测。
2. 连续视觉丢失：降低速度并搜索目标。
3. 定位协方差过大：禁止高速运动。
4. 编码器或IMU异常：进入受限模式或停车。
5. 轨迹超时：沿当前速度受控减速，不应突然清零PWM。

### 7.3 评价指标

感知与规划层可使用以下指标评价：

| 类别 | 指标 |
|------|------|
| 感知 | 检出率、误检率、位置误差、处理延迟 |
| 定位 | 平均误差、最大误差、漂移率、协方差一致性 |
| 路径 | 路径长度、最小障碍距离、最大曲率 |
| 轨迹 | 最大速度、加速度、jerk、重规划连续性 |
| 系统 | 任务完成时间、成功率、碰撞次数、停车误差 |

## 八、电赛典型方案

### 8.1 规则赛道巡线

```text
摄像头或红外阵列
      ↓
路径中心线/横向误差/航向误差
      ↓
短期路径拟合
      ↓
基于曲率限速 + S曲线速度规划
      ↓
Pure Pursuit、Stanley或横向误差控制器
      ↓
底盘速度环
```

这种场景通常不需要完整SLAM和A*，重点是低延迟视觉、稳定的中心线拟合、合理的前视距离和弯道限速。

### 8.2 已知场地图导航

```text
编码器 + IMU + AprilTag/UWB
      ↓
EKF状态估计
      ↓
已知栅格或拓扑地图
      ↓
A*全局规划
      ↓
路径平滑和速度规划
      ↓
局部避障或轨迹跟踪
```

### 8.3 视觉目标接近

```python
class VisualServo:
    def compute_control(self, target):
        if target is None:
            return 0.0, 0.3  # 低速旋转搜索目标

        image_error = target.center_x - target.image_center_x
        distance_error = target.distance - target.desired_distance

        linear_velocity = clamp(0.8 * distance_error, 0.0, 0.5)
        angular_velocity = clamp(-0.01 * image_error, -1.0, 1.0)
        return linear_velocity, angular_velocity
```

若目标位置变化明显，应该先通过状态估计或目标跟踪滤波预测目标运动，避免直接对有噪声的单帧检测结果进行控制。

## 九、推荐的实施顺序

对于电赛小车，不建议一开始就堆叠SLAM、深度网络和MPC。更稳妥的开发顺序是：

1. 明确比赛任务、地图是否已知、障碍物是否动态。
2. 定义坐标系、时间戳、数据单位和模块接口。
3. 完成编码器与IMU的基础状态估计。
4. 完成最小可用的视觉或红外感知。
5. 用记录数据离线验证感知精度和延迟。
6. 实现简单状态机和固定路径任务。
7. 加入路径平滑、曲率限速和S曲线速度规划。
8. 根据需要加入A*、局部避障或视觉地标校正。
9. 最后再优化高级算法和计算性能。

整个架构的关键不是选择最复杂的算法，而是保证信息链条完整：

```text
可靠观测 → 一致状态 → 可执行路径 → 连续轨迹 → 稳定控制
```

其中任一环节缺少坐标、时间或约束定义，最终都可能表现为底盘抖动、转向迟滞、轨迹不连续或控制器难以调参。
