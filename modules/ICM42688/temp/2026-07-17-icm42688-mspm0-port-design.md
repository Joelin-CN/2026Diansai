# ICM42688 MSPM0G3507 驱动移植设计

## 1. 目标与范围

将 `E:\B306\科研项目\陀螺仪\Gyro20260123\ICM` 中原有的 STM32 ICM42688 驱动及 AHRS 姿态解算代码迁移到 MSPM0G3507 LQFP-48，使用 TI MSPM0 DriverLib，并放置在 `modules/ICM42688` 下。

本次工作包含：

- ICM42688 SPI 寄存器驱动及物理量转换。
- 加速度计与陀螺仪配置、读取和校准能力。
- 原有 Mahony AHRS 六轴姿态解算。
- MSPM0G3507 的 SPI、GPIO、延时和微秒时间接口。
- 模块根目录 `README.md`，说明硬件连接、工程集成、SysConfig 配置和 FreeRTOS 注意事项。

本次工作不修改 `controller/` 下的任何文件。需要在控制器工程中增加或修改的内容统一写入 `README.md`，由使用者集成。

## 2. 硬件约束

目标器件为 MSPM0G3507 LQFP-48，ICM42688 使用 SPI1：

| 信号 | MSPM0 引脚 | 方向 |
| --- | --- | --- |
| SPI1_POCI | PA16 | ICM42688 到 MSPM0 |
| SPI1_SCLK | PA17 | MSPM0 到 ICM42688 |
| SPI1_PICO | PA18 | MSPM0 到 ICM42688 |
| ICM42688_CS | PB20 | MSPM0 到 ICM42688，低有效 |

SPI 使用 Mode 0、8 位数据、MSB first，初始速率为 1 MHz。片选由 PB20 作为普通 GPIO 软件控制。

## 3. 方案选择

采用最小修改的分层移植方案：保留原驱动的平台无关核心和 AHRS 算法，仅替换 STM32 硬件接口层。

未采用单文件扁平化方案，因为它会把 DriverLib 细节与传感器、算法逻辑混合。未在本次直接修改 SysConfig 或控制器工程，因为使用者明确要求 `controller/` 保持不变。

职责划分：

- `icm42688_hal.*`：寄存器定义、初始化、量程与输出数据率配置、原始数据读取和物理量转换，通过通信与延时函数指针访问平台接口。
- `ahrs_hal.*`：Mahony 六轴姿态融合、四元数更新和欧拉角输出，不直接访问硬件。
- `icm42688_mspm0.*`：使用 TI DriverLib 完成 SPI1 阻塞收发、PB20 片选、毫秒延时和 TIMG0 微秒计时。
- `README.md`：唯一的最终说明文档，包含所有 `controller/` 工程侧集成步骤。

## 5. 通信与初始化流程

寄存器读取流程：

1. 拉低 PB20。
2. 发送设置 bit 7 后的寄存器地址。
3. 发送空字节并读取返回数据；连续读取时保持片选有效。
4. 拉高 PB20。

寄存器写入流程：

1. 拉低 PB20。
2. 发送清除 bit 7 后的寄存器地址。
3. 发送数据字节。
4. 拉高 PB20。

传感器初始化沿用原驱动序列：等待上电、读取 `WHO_AM_I` 并校验 `0x47`、软复位、唤醒加速度计与陀螺仪、设置量程和输出数据率、配置滤波器并等待稳定。

SPI 外设初始化由控制器工程的 SysConfig 负责。模块只使用已经初始化的 SPI1，不重复修改 SPI1 时钟或引脚复用。

## 6. 微秒计时与 FreeRTOS 兼容

AHRS 的采样间隔使用 TIMG0 生成。TIMG0 配置为 1 MHz 自由运行计数器，读取计数值即可获得 32 位微秒时间戳。时间差使用无符号减法计算，使一次计数回绕仍可正确处理。

FreeRTOS 在 MSPM0 Cortex-M0+ 端口中通常占用 SysTick 作为系统节拍，因此驱动不使用 SysTick。当前项目的 TIMA0 和 TIMA1 已用于电机 PWM，TIMG0 与其独立。

为遵守不修改 `controller/` 的约束，模块不直接添加 SysConfig 配置。`README.md` 将明确说明如何在 SysConfig 中配置 TIMG0，以及必须使用的实例名、1 MHz 计数频率和自由运行模式。驱动的平台适配头允许将工程生成的 SPI、GPIO 和 TIMG0 符号映射到模块所需名称。

## 7. AHRS 数据流

应用按固定周期执行：

1. 读取一次连续的加速度计和陀螺仪数据。
2. 将原始大端 16 位数据转换为 `g` 和 `dps`。
3. 读取当前 TIMG0 微秒计数，与上次时间戳相减并换算为秒。
4. 将六轴数据和实际 `dt` 输入 Mahony 更新函数。
5. 按需读取 roll、pitch、yaw。

陀螺仪输入 AHRS 前按算法要求由度每秒转换为弧度每秒。偏置校准只能在传感器静止时执行；README 中会明确该前置条件。

六轴 AHRS 无法仅依靠加速度计和陀螺仪长期约束绝对航向，因此 yaw 允许随陀螺仪零偏缓慢漂移，文档不得将其描述为绝对航向。

## 8. 错误处理

- 初始化返回明确状态，至少区分成功和设备身份校验失败。
- 公共读取接口校验输出指针，避免空指针写入。
- SPI 使用阻塞轮询，并设置有限等待或检查 DriverLib 状态，避免硬件异常时永久卡死。
- AHRS 遇到无效加速度向量时跳过归一化反馈，但仍可执行陀螺仪积分。
- 对明显异常的 `dt` 不进行姿态积分，防止暂停或调试断点导致单次姿态突变。

## 9. 测试与验证

由于目标模块依赖 MSPM0 DriverLib 和实物传感器，验证分为两层：

- 主机侧测试：验证寄存器地址读写位、原始大端数据解析、量程换算、计时回绕差值和 AHRS 数学输出；通过伪造平台接口隔离硬件。
- 目标板验证：编译 MSPM0 工程，读取 `WHO_AM_I == 0x47`，静止时检查加速度模长接近 1 g、陀螺仪校准后接近零，并转动板卡检查 roll/pitch 方向和幅值。

若现有工程没有可直接运行的主机测试框架，则至少执行 AC6 编译检查，并把需要上板确认的项目列入 README。
