# 小车底盘控制系统完整设计框架

## 概念澄清：前馈 ≠ 前视传感器

常见的误解是把"前馈"理解成了"摄像头等前视传感器"，实际上：

| 术语 | 正确含义 | 举例 |
|------|----------|------|
| **前馈控制（Feedforward）** | 基于**期望轨迹/模型**的开环控制，不依赖传感器反馈 | 根据轨迹规划计算出理论电机转速 |
| **反馈控制（Feedback）** | 基于**传感器测量误差**的闭环控制 | PID根据编码器误差调整PWM |
| **感知层** | 摄像头、激光雷达等**环境感知** | 视觉识别路径、避障 |
| **状态估计层** | 编码器、IMU等**自身状态测量** | 里程计、姿态融合 |

**正确的架构**应该是：

```
感知层（摄像头）→ 路径规划 → 前馈控制 ─┐
                                      ├→ 执行器（电机）
状态估计（编码器/IMU）→ 反馈控制 ────┘
```

### 关键模块详解

#### 1. **前馈控制器**（开环，快速响应）

**作用**：根据期望轨迹提前计算理论控制量，减轻反馈负担

**组成**：
```python
# 速度前馈（逆运动学）
v_target, omega_target = trajectory.get_velocity(t)
v_L_ff = v_target - b/2 * omega_target
v_R_ff = v_target + b/2 * omega_target

# 加速度前馈（动力学补偿）
a_target = trajectory.get_acceleration(t)
tau_ff = J * a_target  # 惯性补偿

# 摩擦补偿
tau_friction = k_friction * sign(v_target)

# 总前馈
u_ff = K_v * v_ff + K_a * a_ff + tau_friction
```

#### 2. **反馈控制器**（闭环，鲁棒修正）

**作用**：根据实际误差实时修正，补偿模型误差和扰动

**候选方案**：

| 控制器 | 优点 | 缺点 | 适用场景 |
|--------|------|------|----------|
| **PID** | 简单、稳定、易调参 | 无法处理约束、预测能力弱 | 低速、简单任务 |
| **串级PID** | 抗扰能力强、响应快 | 调参复杂 | 需要高性能电流环 |
| **LQR** | 最优控制、系统化设计 | 需要精确模型、无约束处理 | 倒立摆、平衡车 |
| **MPC** | 预测能力、处理约束 | 计算量大 | 高速路径跟踪 |
| **滑模控制** | 鲁棒性极强 | 抖振问题 | 强扰动环境 |
| **自适应控制** | 在线学习参数 | 收敛慢、调参难 | 参数时变系统 |

**推荐组合**：
- **入门**：单环PID（速度环）
- **进阶**：串级PID（外环速度 + 内环电流）
- **高级**：MPC（路径跟踪）+ 内环PID（电机控制）

#### 3. **状态估计器**

融合多传感器数据，得到准确的当前状态：

```python
# 扩展卡尔曼滤波（EKF）
# 预测步
x_pred = f(x_prev, u)  # 根据运动学模型预测
P_pred = F * P * F.T + Q

# 更新步（融合编码器、IMU）
y = z - h(x_pred)  # 观测残差
K = P_pred * H.T * inv(H * P_pred * H.T + R)
x_est = x_pred + K * y
P_est = (I - K * H) * P_pred
```

## 问题1：如何测算系统极限

### 1.1 震荡幅度与平滑度指标

**时域指标**：

| 指标 | 定义 | 目标值 |
|------|------|--------|
| **超调量** | $\sigma = \frac{y_{\max} - y_{\text{steady}}}{y_{\text{steady}}} \times 100\%$ | < 5% |
| **调节时间** | 误差进入±2%稳态范围的时间 | < 0.5s |
| **稳态误差** | $e_{ss} = \lim_{t \to \infty} (r(t) - y(t))$ | < 1 mm 或 1° |
| **速度波动** | $\text{std}(v_{\text{measured}})$ | < 5% 目标速度 |

**频域指标**：

| 指标 | 定义 | 含义 |
|------|------|------|
| **截止频率** | 开环增益为1时的频率 | 系统响应速度 |
| **相位裕度** | $PM = 180° + \angle G(j\omega_c)$ | 稳定性裕量（> 45°） |
| **增益裕度** | $GM = -20\log|G(j\omega_{180°})|$ | 抗扰能力（> 6 dB） |

## 问题2：PID之外的选择

### 2.1 控制器对比

#### **模型预测控制（MPC）**
```python
# 优势：考虑未来N步，处理约束
def mpc_controller(x_current, trajectory, N=10):
    cost = 0
    for i in range(N):
        x_pred = predict(x_current, u[i])
        x_ref = trajectory[i]
        cost += ||x_pred - x_ref||^2 + ||u[i]||^2
        
        # 约束
        subject_to:
            |u[i]| <= u_max
            |v[i]| <= v_max
    
    u_opt = minimize(cost)
    return u_opt[0]  # 只执行第一步
```

**何时用MPC**：
- 需要预测未来轨迹（高速避障）
- 有硬约束（速度/加速度限制）
- 计算资源充足（至少50 Hz）

#### **LQR（线性二次型调节器）**
```python
# 优势：理论最优、系统化调参
A, B = linearize_dynamics(x0, u0)
Q = diag([100, 100, 10, 1, 1])  # 状态权重
R = diag([1, 1])                # 控制权重

K = lqr(A, B, Q, R)
u = -K @ (x - x_ref)
```

**何时用LQR**：
- 系统线性或可线性化
- 追求理论最优
- 不需要硬约束

#### **自适应律（如RBF神经网络）**
```python
# 在线学习系统参数
def adaptive_controller(x, x_ref):
    u_nominal = pid(x, x_ref)
    u_adaptive = rbf_network.predict(x)  # 学习非线性补偿
    rbf_network.update(error)  # 在线更新权重
    return u_nominal + u_adaptive
```

## 问题3：实际设计流程

### Step 1: 系统建模
```python
# 差速底盘运动学
def kinematics(v, omega, b):
    v_L = v - b/2 * omega
    v_R = v + b/2 * omega
    return v_L, v_R

# 电机动力学
def motor_dynamics(V, omega, J, b, Kt, Ke, R):
    tau = Kt * (V - Ke*omega) / R
    domega_dt = (tau - b*omega) / J
    return domega_dt
```

### Step 2: 控制器设计
```python
class ChassisController:
    def __init__(self):
        self.pid_left = PID(Kp=1.5, Ki=0.3, Kd=0.05)
        self.pid_right = PID(Kp=1.5, Ki=0.3, Kd=0.05)
        
    def compute(self, v_target, omega_target, v_actual, omega_actual):
        # 前馈：逆运动学
        v_L_ff, v_R_ff = self.inverse_kinematics(v_target, omega_target)
        
        # 反馈：PID
        v_L_actual, v_R_actual = self.inverse_kinematics(v_actual, omega_actual)
        u_L_fb = self.pid_left.compute(v_L_ff, v_L_actual)
        u_R_fb = self.pid_right.compute(v_R_ff, v_R_actual)
        
        # 合成
        u_L = self.K_ff * v_L_ff + u_L_fb
        u_R = self.K_ff * v_R_ff + u_R_fb
        return u_L, u_R
```
